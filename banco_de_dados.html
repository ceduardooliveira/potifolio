Banco de Dados Relacionais

Nesta máteria aprendi sobre banco de dados relacionais. Esses dados sã ochamados de realacionais porque as tabelas dentro do banco de dados podem estar relacionadas entre si por meio de chaves primárias e estrangeiras.

Dentro da estrutrura, temos: Tabelas, onde tem linhas (registros) e colunas (atributos). Chave primária, é uma coluna ou um conjuto de colunas que identifica unicamente cada linha em uma tabela. Chave estrangeira, é uma coluna ou conjuto de colunas que estabele um vínculo entre duas tabelas, referenciando a chave primária de outra tabela.

A normalização é o processo de organizar os dados para reduzir a redundância e melhorar a integridade dos dados.

Tem a operações em banco de dados.

    CRUD: As operações básicas de banco de dados são CREATE(criar), READ(ler), UPDATE(atualizar), e DELETE(excluir).
    SQL (Structured Query Language): A linguagem padrão para manipulação de dados relacionais, usada para executar consultas, inserir, atualizar e excluir dados.

Ao lado o modelo Entidade Relacional.

    Neste moledo ER, Uma biblioteca deseja informatizar seu sistema de gestão de livros. Cada livro tem um título, um ISBN (International Standard Book Number), uma editora e um ano de publicação. Cada livro pode ter vários autores, mas cada autor pode ter escrito vários livros. Modele este sistema utilizando um diagrama de entidade-relacionamento.
    Na entidade LIVRO temos ISBN como chave primária e TITULO, EDITORA, ANO_PUBLICADO como atributos. Do outro lado temos a entidade AUTOR, com ID_AUTOR chave primária e NOME como atributo. Entre essas duas entidades, temos a entidade ESCRITO com as chaves estrangeiras ID_AUTOR e ISBN. Neste caso podemos fazer a seguinte leitura, que LIVROS pode ter varios AUTOR e AUTOR pode ter escrito vários LIVROS.

Na segunda imagem tem mais um modelo ER.

    Na entidade PROJETO temos o ID_PROJETO como chave primária e NOME_PROJETO, DESCRIÇÃO como atributos. Do outro lado temos a entidade FUNCIONÁRIOS que tem como chave primária o ID_FUNCIONARIO e como atributo NOME_FUNCIONARIO, DATA_ADMISSÃO. Entres essas duas entidades, temos a entidade POSSUI, com duas chaves estrangeiras, ID_PROJETO e ID_FUNCIONARIO. Neste caso o relacionamento delas são N para N, onde PROJETO possui vários FUNCIONARIOS, e Funcionarios possui vários PROJETOS.

Diagrama

Comandos DDL, DML, DQL.

Diagrama DDL - Data Definition Language. São os seguintes comandos.

    CREATE TABLE - ALTER TABLE - DROP TABLE

Comando DML - Data Manipulation Language. São os seguintes comandos.

    INSERT - UPDATE - DELETE - SELECT

Comando DQL - Data Query Language. São os seguintes comandos.

    SELECT - OPERADORES SQL - ORDER BY - JOIN - GROUP BY

Parte 1

Nesta matéria aprendi a programar o arduino usando a linguagem de programação C++. Para programar o arduino usamos o site TINKECAD. Fizemos varias atividades para treinar a lógica de programação, e varios tipos de montagem, esquemas, aplicações para testar o funcionamento desta placa eletrônica, tais como: temporizadores, contagem do 1 ao 9, botões para acionamento.

Na primeira figura ao lado é um arduino simulando um semáfaro de carro e semáfora de pedestres onde tivemos a seguinte lógica implementada:

1. Passo: Declaração das variáveis.

As variáveis vermelho, amarelo, e verde representam os pinos onde os LEDs do semáforo de carro estão conectados. As variáveis vermelho_pedestre e verde_pedestre são para os LEDs do semáforo de pedestre. botao é o pino onde o botão de pedestre está conectado.

2. Passo: Configuação Inicial.

No setup(), os pinos são configurados como entradas (INPUT) ou saídas (OUTPUT). Em seguida, os LEDs verde do carro (verde) e vermelho do pedestre (vermelho_pedestre) são ligados (HIGH), configurando o estado inicial do semáforo.

3. Passo: Loop principal:

No loop(), o código verifica se o botão de pedestre foi pressionado (digitalRead(botao) == LOW). Se o botão for pressionado, o seguinte ocorre:

1. Transição para amarelo: O LED verde do carro (verde) é desligado (LOW) e o LED amarelo (amarelo) é ligado (HIGH). O semáforo permanece amarelo por 5 segundos (delay(5000)).

2. Transição para vermelho: O LED amarelo é desligado (LOW) e o LED vermelho do carro (vermelho) é ligado (HIGH). Após 1 segundo (delay(1000)), o LED vermelho do pedestre (vermelho_pedestre) é desligado (LOW) e o LED verde do pedestre (verde_pedestre) é ligado (HIGH). O semáforo de pedestre fica verde por 5 segundos (delay(5000)).

3. Aviso para os pedestres: O LED verde do pedestre é desligado (LOW) e o LED vermelho do pedestre começa a piscar por 5 vezes, com intervalos de 0,5 segundo (delay(500)) entre cada piscada.

4. Finalização: O LED vermelho do pedestre é ligado novamente (HIGH) e, após 1 segundo (delay(1000)), o LED vermelho do carro é desligado (LOW) e o LED verde do carro é ligado (HIGH), retornando ao estado inicial do semáforo de carro.

Este ciclo se repete sempre que o botão de pedestre é pressionado.

Parte 2

Nesta segunda parte usamos o site Wokwi e apredemos a usar o ESP32, utilizamos a linguagem de programação C++, esse disposito mostra o resultado esperado na tela do seu visor.

Neste projeto ao lado continha um menu para acessar o que desejava fazer, botões para avançar e iniciar a função, tais como:

1. Relógio. 2. Contador. 3. Limpar tela.

No exemplo ao lado está o códio do relogio.

A função mostra_relogio(h, m, s, false) é chamada para exibir as horas, minutos e segundos. Em seguida, o código aguarda por um tempo determinado pela variável tempo.

Tem a verificação do estado dos botões bt_azul e bt_amarelo. Se bt_azul estiver pressionado (digitalRead(bt_azul) == 1), o tempo de atraso (tempo) é definido como 250 milissegundos. Se bt_amarelo estiver pressionado, tempo é definido como 1 milissegundo. Isso permite ajustar a velocidade com que os segundos são incrementados.

O código simula a passagem do tempo de um relógio, começando de 0 horas e 18 minutos, incrementando os segundos até 59, depois incrementando os minutos e eventualmente as horas. A velocidade da simulação pode ser ajustada usando dois botões. bt_azul ajusta o atraso entre as atualizações para 250 milissegundos e bt_amarelo ajusta o atraso para 1 milissegundo, tornando o relógio muito mais rápido.
